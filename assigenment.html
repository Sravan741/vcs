GitHub Commands - Detailed Definitions

  • init: The git init command initializes a new Git repository in the current directory. It creates a hidden .git directory that contains all the metadata and structure required for Git to track the project. This command is the first step when starting to use Git for a new project.

    Example: git init creates a repository, making the directory ready to track file changes.

  • add: The git add command stages changes in specified files or directories. These changes are queued to be included in the next commit. Without staging changes, they won't be saved permanently to the repository.

    Example: git add filename stages a specific file, while git add foldername stages all files within a folder.

  • add .: A shorthand for staging all changes in the current directory and its subdirectories. This includes modified, newly created, or deleted files. It is a convenient way to prepare all changes for a commit.

    Example: git add . stages every file that has been changed, added, or deleted in the current project directory.

  • status: The git status command provides an overview of the repository's current state. It shows which files are staged for commit, which are modified but unstaged, and which files are untracked (new files not added to the repository).

    Example: git status outputs detailed information about file states, helping track work in progress.

  • log: The git log command displays the commit history of the repository. Each commit is listed with its unique identifier (SHA hash), author name, date, and commit message. This helps in understanding the changes and their order.

    Example: git log shows a list of all commits, while git log --oneline displays a simplified view.

  • reset: The git reset command is used to undo changes in a repository. It can unstage files from the staging area (git reset) or revert the repository to a specific previous commit, discarding subsequent changes (git reset --hard).

    Example: git reset HEAD filename unstages a file, while git reset --hard commit-hash resets the repository to a specific commit.

  • pull: The git pull command fetches the latest changes from a remote repository and integrates them into the current branch. It is a combination of two commands: git fetch (to retrieve changes) and git merge (to integrate them).

    Example: git pull origin main fetches and merges updates from the main branch of the remote repository.

  • push: The git push command uploads local commits to a remote repository. It updates the remote branch with the latest changes made locally, allowing team members to access them.

    Example: git push origin main pushes the changes from the local main branch to the corresponding branch in the remote repository.

  • branch: The git branch command is used to manage branches in a Git repository. It can create new branches, list existing branches, or delete branches. Branches allow working on features or bug fixes independently without affecting the main codebase.

    Examples:

    • git branch: Lists all branches.
    • git branch branch-name: Creates a new branch.
    • git branch -d branch-name: Deletes a branch.
